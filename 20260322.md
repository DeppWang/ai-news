# AI 新基建：从百万上下文到机器支付

## 1. Claude 百万 Token 上下文窗口正式全面开放

**来源：** [https://claude.com/blog/1m-context-ga](https://claude.com/blog/1m-context-ga)

Anthropic 宣布 Claude Opus 4.6 和 Sonnet 4.6 全面开放 100 万 Token 上下文窗口，且不再收取长上下文溢价。Opus 4.6 定价为每百万 Token 输入 5 美元 / 输出 25 美元，Sonnet 4.6 为 3 美元 / 15 美元，无论请求长度如何，单价保持一致。

在媒体处理方面，单次请求最多支持 600 张图片或 PDF 页面，较此前的 100 项限制提升了 6 倍。Opus 4.6 在 MRCR v2 基准测试中达到 78.3% 的准确率，在整个百万 Token 窗口内保持稳定的召回能力。Max、Team 和 Enterprise 用户的 Claude Code 将自动启用完整上下文窗口，减少对话压缩事件，开发者可以直接加载完整代码库或长文档，无需有损摘要。

## 2. Stripe 推出机器支付协议，为 AI 智能体构建交易基础设施

**来源：** [https://stripe.com/blog/machine-payments-protocol](https://stripe.com/blog/machine-payments-protocol)

Stripe 与 Tempo 联合推出机器支付协议（Machine Payments Protocol，MPP），这是一个开放标准，允许 AI 智能体在无需人类干预的情况下自主完成支付交易。当前支付系统要求类似人类的交互流程 —— 创建账户、浏览定价页、选择订阅、输入支付信息，这些步骤对自主智能体构成了巨大障碍。

MPP 的工作流程非常简洁：智能体向服务发起资源请求，服务返回支付要求，智能体授权并完成支付，资源随即交付。开发者只需几行代码即可通过 Stripe 的 PaymentIntents API 完成集成，支持稳定币、法币卡和先买后付等多种支付方式。早期合作伙伴包括 Browserbase（按会话付费的浏览器基础设施）和 PostalForm（程序化实体邮件服务）等，标志着智能体经济基础设施的初步成型。

## 3. ICML 因审稿违规使用 LLM 拒绝 2% 论文

**来源：** [https://blog.icml.cc/2026/03/18/on-violations-of-llm-review-policies/](https://blog.icml.cc/2026/03/18/on-violations-of-llm-review-policies/)

机器学习顶级会议 ICML 对审稿过程中违规使用大语言模型的行为采取了执法行动。大会实施了双轨政策框架，允许审稿人在保守模式（禁止使用 LLM）和宽松模式（有限使用 LLM 辅助）之间选择，但仍有大量审稿人违反了约定。

最终共有 497 篇论文（约占投稿总量的 2%）因此被拒，795 篇审稿意见（约占全部审稿的 1%）被标记为违规，涉及 506 名审稿人，其中 51 人因系统性违规被彻底移出审稿池。ICML 的检测方法颇具巧思：在提交的 PDF 中嵌入隐藏的 LLM 指令水印，从 17 万个词汇的词典中随机选取两个短语，AI 系统会自动响应这些指令而人类审稿人则不会察觉。该方法在测试中检测成功率超过 80%，且每个被标记的实例都经过人工复核以防止误判。

## 4. Mistral 发布 Forge 企业级 AI 模型开发平台

**来源：** [https://mistral.ai/news/forge](https://mistral.ai/news/forge)

Mistral AI 推出 Forge 平台，帮助企业基于自有专有数据构建前沿级 AI 模型。该平台支持三个训练阶段：在内部数据集上进行预训练以建立领域认知，通过后训练针对特定任务优化行为，以及利用强化学习使模型与组织策略和运营目标对齐。

Forge 在架构上同时支持稠密模型和混合专家（MoE）架构，并支持文本、图像等多模态输入。平台采用智能体优先设计，代码智能体可以利用 Forge 自动微调模型、优化超参数和生成合成训练数据。企业对模型、数据和知识产权拥有完全控制权，这对金融、政府等受监管行业尤为关键。目前 ASML、欧洲航天局、爱立信等机构已成为首批合作伙伴。

## 5. Chrome DevTools MCP：让 AI 编程助手直接调试浏览器

**来源：** [https://developer.chrome.com/blog/chrome-devtools-mcp-debug-your-browser-session](https://developer.chrome.com/blog/chrome-devtools-mcp-debug-your-browser-session)

Google 推出 Chrome DevTools MCP 服务器，允许 AI 编程助手通过模型上下文协议（MCP）直接连接并调试活跃的 Chrome 浏览器会话。该功能的核心优势在于会话复用 —— 智能体可以利用用户已登录的浏览器会话，无需额外认证即可访问需要登录的页面进行调试。

该功能将在 Chrome M144 Beta 中推出，采用基于权限的远程调试流程：用户需在 chrome://inspect/#remote-debugging 中显式启用远程调试，MCP 服务器发起连接时 Chrome 会弹出权限对话框要求用户确认，调试会话期间浏览器会持续显示自动控制指示条。开发者可以在 DevTools 中选中某个有问题的网络请求或 DOM 元素，然后直接让 AI 助手针对该上下文进行调查分析，实现手动调试与 AI 辅助的无缝切换。
