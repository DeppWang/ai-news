## 2. MegaTrain：在单张 GPU 上全精度训练 1000 亿参数大模型

![](https://arxiv.org/static/browse/0.3.4/images/arxiv-logo-fb.png)

**来源：** [https://arxiv.org/abs/2604.05091](https://arxiv.org/abs/2604.05091)

**摘要：** MegaTrain 是一个以内存为中心的训练系统，将参数和优化器状态存储在 CPU 内存中，把 GPU 作为瞬时计算引擎。系统采用流水线双缓冲执行机制，重叠参数预取、计算和梯度卸载，并使用无状态层模板消除自动求导图的内存开销。在单张 H200 GPU 配合 1.5TB 主机内存的环境下，MegaTrain 成功以全精度训练高达 1200 亿参数的模型，在训练 140 亿参数模型时吞吐量达到 DeepSpeed ZeRO-3 的 1.84 倍。这一范式转变让此前需要多 GPU 集群才能完成的大模型训练变得更加民主化。

## 4. Claude Code 技能：让 AI 说原始人语言大幅削减 token 用量

**来源：** [https://github.com/JuliusBrussee/caveman](https://github.com/JuliusBrussee/caveman)

**摘要：** Caveman 是一个获得 347 个 HN 点赞的 Claude Code 技能插件，通过让 AI 以电报式精简语言回复来大幅减少 token 消耗。该工具支持 40 多个 AI 助手，包括 Claude Code、GitHub Copilot 和 Cursor。实测显示可节省 65-75% 的输出 token，同时保持技术准确性。例如，原本冗长的解释会被压缩为简洁的技术片段。该插件提供 Lite、Full、Ultra 和文言文四个精简级别，还包含精简提交信息和单行代码审查等专用技能。基准测试表明，在多种任务中始终能节省 22-87% 的 token，且不牺牲准确性。

![](https://cdn.sanity.io/images/4zrzovbb/website/1f6efd2520ecc6bc7b5bccd6f4bbcf79bb47dbc1-3764x2380.png)

## 20. 八年心愿三个月实现：AI 辅助构建 SQLite 开发工具的启示

**来源：** [https://lalitm.com/post/building-syntaqlite-ai/](https://lalitm.com/post/building-syntaqlite-ai/)

**摘要：** 开发者 Lalit Maganti 分享了利用 AI 在三个月内完成了憧憬八年之久的 SQLite 开发工具套件 syntaqlite 的经历。该工具包含从 SQLite 源代码提取的高保真解析器、代码格式化器、linter、LSP 实现和 VS Code 扩展。AI 在标准代码生成、加速学习新领域知识和克服启动惰性方面表现出色，但在设计决策、架构选择和 API 人机工程学方面存在明显不足。关键教训是：当作者在早期放弃代码掌控进行纯 AI 编程时，复杂性失控需要完全重写。专业领域知识决定了 AI 的有效性 ——AI 在有深厚领域知识的情况下表现优异，但在需要探索的场景中容易失败。
